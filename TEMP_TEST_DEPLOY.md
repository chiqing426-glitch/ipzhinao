# IP智脑临时测试部署方案

## 推荐方案：云端测试链接

本地临时外链会跟随电脑睡眠、重启、断网而失效。要连续开放几天给 100 人测试，建议把当前 MVP 放到云端测试平台。

推荐优先级：

1. Railway：适合快速测试，支持 GitHub 自动部署，临时域名可直接分享，适合先跑几天看反馈。
2. Render：也能快速部署，但免费 Web 服务空闲后会休眠，且本地 SQLite 数据会在重启/重新部署后丢失；如果要保存数据，建议用 Starter 实例加磁盘，或改接 Postgres。
3. Zeabur：国内用户体验可能更顺，但通常需要绑定自己的服务器或订阅后使用完整能力。

## 你需要做的事

1. 注册一个 GitHub 账号，创建一个私有仓库。
2. 注册 Railway 或 Render。
3. 授权平台读取这个私有仓库。
4. 配置环境变量：

```text
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_MODEL=deepseek-v4-flash
FREE_AI_LIMIT=3
HOST=0.0.0.0
DB_PATH=平台持久化磁盘路径/app.db
```
